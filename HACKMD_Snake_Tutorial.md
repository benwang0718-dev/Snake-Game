# 進階版貪吃蛇教學

## 專案目標

這個版本的貪吃蛇是參考 `Unit04` 的分層方式製作，但功能更完整，適合拿來當作期末作業、專題練習或 HackMD 教學範例。

功能包含：

- PyQt6 視窗化介面
- 鍵盤控制上下左右移動
- 計分系統
- 最高分保存
- 關卡提升
- 障礙物機制
- 特殊加分食物
- 暫停、重新開始、離開遊戲

## 專案結構

```text
game/
├─ __init__.py
├─ main.py
├─ requirements.txt
├─ HACKMD_Snake_Tutorial.md
├─ high_score.json
├─ config/
│  └─ app_config.py
├─ services/
│  ├─ game_service.py
│  └─ score_service.py
└─ ui/
   ├─ helpers.py
   └─ main_window.py
```

## 安裝方式

### 1. 安裝套件

```bash
pip install -r game/requirements.txt
```

### 2. 啟動遊戲

```bash
python -m game.main
```

也可以直接執行：

```bash
python game/main.py
```

## 操作方式

- `WASD` 或方向鍵：移動蛇
- `P`：暫停 / 繼續
- `R`：重新開始
- `Q` 或 `Esc`：離開遊戲

## 為什麼要參考 Unit04

`Unit04` 已經展示了很好的初學者專案分層：

- `config` 放設定值
- `services` 放遊戲邏輯
- `ui` 放畫面
- `main.py` 當程式入口

這樣做的好處是：

- 程式更容易維護
- UI 和邏輯分開，比較清楚
- 後續加功能時不容易亂掉

## 各檔案說明

### `game/config/app_config.py`

這裡集中管理所有設定，例如：

- 遊戲視窗標題
- 地圖大小
- 每格大小
- 初始速度
- 升級規則
- 分數規則
- 顏色設定
- 最高分檔案路徑

這樣之後想改速度、版面、分數都只要改這一個地方。

### `game/services/game_service.py`

這是核心邏輯層，負責：

- 建立初始蛇身
- 處理移動方向
- 判斷是否撞牆
- 判斷是否撞到自己
- 判斷是否吃到食物
- 產生障礙物
- 管理關卡和加速
- 管理特殊食物倒數

這一層不直接碰 UI，所以測試起來也比較方便。

### `game/services/score_service.py`

這一層專門處理最高分資料：

- 讀取 `high_score.json`
- 寫入新的最高分

把資料儲存獨立出去，是很常見的實務做法。

### `game/ui/helpers.py`

這個檔案負責把畫面上方狀態列的字串整理好，例如：

- 目前分數
- 最高分
- 關卡
- 速度
- 特殊食物剩餘時間

### `game/ui/main_window.py`

這是 PyQt6 視窗主體，負責：

- 建立視窗
- 接收鍵盤事件
- 啟動計時器
- 每個 tick 更新遊戲
- 繪製格線、蛇、食物、障礙物、狀態列

## 遊戲流程解說

### 1. 建立視窗

在 `main.py` 中建立 `QApplication`，再把 `SnakeWindow` 顯示出來。

### 2. 初始化遊戲資料

`create_new_game()` 會建立：

- 初始蛇身
- 起始方向
- 一般食物
- 關卡
- 障礙物
- 初始速度
- 分數

### 3. 使用 `QTimer` 讓遊戲持續更新

PyQt6 的 `QTimer` 會固定時間觸發 `on_tick()`，每一次觸發就代表遊戲往前更新一格。

### 4. 在 `advance_game()` 處理核心邏輯

每次更新時會做這些事：

1. 依照方向算出新的蛇頭位置
2. 檢查是否撞牆
3. 檢查是否撞到障礙物
4. 檢查是否撞到自己
5. 檢查是否吃到一般食物
6. 檢查是否吃到特殊食物
7. 必要時升級與加速
8. 更新最高分

## 複雜功能設計重點

### 關卡系統

每吃到一定數量的食物後，`level` 會提升。

升級後會發生兩件事：

- 遊戲速度變快
- 地圖上出現更多障礙物

### 特殊食物

每隔幾次吃到一般食物，就會生成一次特殊食物。

特殊食物特點：

- 分數更高
- 有時間限制
- 沒吃到就會消失

### 最高分保存

最高分會寫入 `high_score.json`，因此下次開遊戲時還能保留紀錄。
